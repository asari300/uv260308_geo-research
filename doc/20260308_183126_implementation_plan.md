# 実装計画 (Implementation Plan)

## 課題の認識と対応策
1. **リンク表示における `[]` (カッコ)の非表示**
   - Streamlit の `st.dataframe` において、`[地点名](URL)` のマークダウンリンクがバージョンや環境によっては正しく解釈されず、そのまま文字列表記されてしまう事象と推測されます。
   - **解決策**: `app.py` において、マークダウンでの結合をやめます。代わりに `Municipality`（地点名）はプレーンテキストとして表示し、別途 `URL` カラムを `st.column_config.LinkColumn("JMAリンク", display_text="気象庁ページへ")` のように設定して表示するか、`Municipality` カラムを URL そのものにして `display_text` を固定化（"開く" 等）することで、カッコ文字が見えない綺麗な UI に整えます。

2. **「最低」などの指標が選択不可になるケースの解消**
   - **原因**: LazyFrame (`polars.scan_csv`) から `unique()` で指標リストを動的に読み込む実装において、Polars の挙動や、事前に生成された `weather_data.csv` の並び順の問題で Streamlit 側のプルダウンに全項目が現れない、または「最初に見つかった最高のみ」に引きずられていた可能性があります。
   - **解決策**: モックデータの指標リスト（最高、最低、日照時間など）を、ジェネレータと `app.py` で共通の定数・リストとして固定管理します。これにより、欠落なくすべての指標が確実にプルダウン（selectbox）から選択できるよう保証します。

3. **上下5位のデフォルト表示と、TSV事前計算による高速化**
   - Streamlit 起動時に毎回すべての csv を読み込みソート処理を行うとオーバーヘッドが生じます。
   - **解決策**: `data_generator.py` の最後で、`polars` を用いて「都道府県別」と「市町村別」それぞれ、全指標について「上位5件」「下位5件」をあらかじめ抽出し、`data/top5_bottom5_{mode}_{metric}.tsv` という形式で保存（事前計算）します。
   - Streamlit (`app.py`) ではユーザーが選択した指標とモードに基づいて、該当の TSV ファイルのみを即座に読み込んで表示します。

4. **ドキュメントの `/doc/` 保存と Playwright MCP 手順書の作成**
   - 思考過程 (`thought`)、Walkthrough、Task を指定されたタイムスタンプ（`YYYYMMDD_HHMMSS`）付きの形式で `/doc/` に保存します（タスクファイルはすでに保存済み）。
   - さらに、ブラウザ自動テストのための **Playwright MCP 導入手順書** を `/doc/` 配下へ作成します。

## 実行フェーズ (EXECUTION) の手順
1. `src/data_generator.py` を改修し、TSV 出力機能（上下5件ずつ）と、指標の明確なリスト化を実装する。
2. `src/app.py` を改修し、CSV ではなく `TSV` から即座にデータを読み込み、UI のリンク表記設定とデータ選択構成をアップデートする。
3. Playwright MCP の設定手順 `.md` ファイルを作成する。
4. すべて完了後、`Walkthrough` や `Thought` を `/doc/` へ配置・作成する。
